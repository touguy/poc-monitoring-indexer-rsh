# Ponder 확장 아키텍처 코드 반영 Walkthrough

사용자님의 승인에 따라, Ponder Core 분석을 기반으로 제안된 4가지 주요 확장 기능(Bloom Filter, Realtime Reorg, Factory Tracking, Traces 수집)을 `poc-monitoring-indexer-rsh` 프로젝트 소스 코드에 모두 성공적으로 적용했습니다.

모든 파일에는 한글 주석을 자세하게 추가하였으며, `🚀 [Ponder 확장 기능]` 태그를 달아 새롭게 추가된 로직임을 명확히 식별할 수 있도록 했습니다.

## 🛠️ 적용된 주요 변경 사항

### 1. Database Schema & Entities
- **`init.sql`**: 
  - 팩토리 패턴 자식 주소 관리를 위한 `dynamic_contracts` 테이블 생성
  - 내부 트랜잭션 기록 보존을 위한 `internal_transaction_records` 테이블 생성
- **Entities & Repositories**: 
  - `DynamicContract`, `InternalTransactionRecord` TypeORM 엔티티 및 Repository 구현 (TypeORM 쿼리빌더 및 일괄 저장 로직 포함)

### 2. (옵션 1) Bloom Filter 최적화
- **`src/common/utils/bloom.util.ts` 신규 생성**: 블록 헤더의 `logsBloom` 필드를 분석하여 불필요한 RPC `eth_getLogs` 호출을 사전에 방지할 수 있는 유틸리티 클래스(`BloomUtil`)를 작성했습니다.
- **`contract-event.service.ts`에 연동**: 블록 수신 시 `BloomUtil.isRelevant` 검사를 통과한 경우에만 필터를 구성하도록 최적화했습니다.

### 3. (옵션 2) Realtime Reorg (실시간 체인 재조직 백트레이싱)
- **`src/block/service/block.service.ts` 수정**: 
  - 인메모리에 `lastProcessedBlockHash`, `lastProcessedBlockNumber` 상태를 캐싱합니다.
  - WebSocket으로 새 블록(`handleNewBlock`)이 인입될 때, 직전 블록의 해시와 현재 블록의 `parentHash`를 대조하여 불일치 시 실시간 역추적(`reorgService.handleReorg`)을 발생시킵니다.
  - 이후 `contractEventService.rollbackEvents`를 호출하여 오염된 상태를 즉시 복원합니다.

### 4. (옵션 3) Dynamic Contracts (팩토리 패턴 동적 주소 수집)
- **`contract-event.service.ts` 수정**: 
  - 초기화(`onModuleInit`) 시점에 DB의 `dynamic_contracts`에서 자식 주소 목록을 불러와 메모리 `Set`에 캐싱합니다.
  - `PairCreated` 혹은 `Created` 이벤트 발생 시 파싱된 로그의 인자를 추출하여 즉시 DB에 영속화하고, 다음 필터링 때 수집 대상(`combinedAddresses`)에 포함되도록 구성했습니다.

### 5. (옵션 4) Internal Transactions (Trace 수집)
- **`blockchain-rpc.service.ts` 수정**: 
  - `debug_traceBlockByHash`를 호출할 수 있도록 래퍼 함수를 신규 구현했습니다.
- **`contract-event.service.ts` 연동**: 
  - `.env`에 `ENABLE_TRACE_SYNC=true`로 설정된 경우, `getLogs`와 `debugTraceBlockByHash`를 `Promise.all`로 병렬 호출합니다.
  - 수집된 Trace 데이터를 `internal_transaction_records` 테이블에 Bulk Insert 하도록 처리했습니다.

## ✅ Verification Results

- **TypeScript 컴파일 및 빌드 테스트**: 
  - `npm run build` 명령을 백그라운드 터미널에서 실행하였으며, 에러나 의존성 충돌 없이 `Exit code: 0`으로 완벽하게 컴파일 됨을 확인했습니다. (모듈 의존성 및 인터페이스 정합성 이상 없음)
- **주석 준수**: 요구사항대로 신규 작성 및 변경된 모든 TS 파일 내 주요 함수와 클래스, 로직 흐름에 걸쳐 아주 상세한 한글 주석을 첨부했습니다.

> [!TIP]
> 이제 `.env`에 `ENABLE_TRACE_SYNC=true` 환경변수만 활성화하시면 노드로부터 내부 트랜잭션 수집까지 자동으로 작동하게 됩니다. (단, 사용 중인 RPC 노드가 Archive/Trace API를 지원해야 합니다.)
