# 문서 구조 개편 완료 (AGENTS.md 분리)

AI 에이전트(Claude 등)와의 최적화된 협업 및 장기 유지보수를 위해, 방대했던 모놀리식 `PRD.md` 문서를 마이크로 문서 단위로 분리하고 이를 연결해주는 최상위 진입점을 구축했습니다.

## 🗂️ 핵심 변경 사항

1. **[NEW] `AGENTS.md` 파일 신규 생성**
   - 개발에 투입되는 AI들이 어떤 파일부터 읽어야 하는지 명시한 인덱스(Index) 맵을 제공합니다.
   - 프로젝트 메타데이터와 핵심 행동수칙(도메인 침범 금지, 주석 작성 의무화 등)을 추가했습니다.

2. **[NEW] `.agent/BUSINESS_LOGIC.md` 로직 분리**
   - 기존 `PRD.md`의 "핵심 비즈니스 로직 상세 설계"(섹션 7) 전체 내용을 단독 파일로 추출했습니다.
   - Initial Sync, 큐 처리, Reorg 감지, 컨트랙트 감지 등 주요 워크플로우를 담당합니다.

3. **[NEW] `.agent/SYSTEM_POLICY.md` 정책 분리**
   - 기존 `PRD.md`의 공통 컴포넌트 정책 및 애플리케이션 환경 변수 스펙(섹션 5, 6)을 단독 파일로 추출했습니다.
   - 예외 처리, 로깅 전략, `.env` 목록이 포함되어 있습니다.

4. **[MODIFY] `PRD.md` 슬라이스 및 링킹 정리**
   - 방대한 내용들이 각각 분리됨으로써 본 문서는 프로젝트 아키텍처 개요와 전체 문서를 연결하는 역할로 요약되었습니다.
   - 가장 상단 주의 사항 블록에 분리된 `.agent/*` 파일들을 모두 올바른 경로로 링킹 처리 완료했습니다.

## 🔎 파일 무결성 및 링크 검증 내용
- `[✅]` `AGENTS.md`에서 각 하위 문서로의 상대 경로 검증 (`./.agent/...`)
- `[✅]` `PRD.md`에서의 상세 문서들 링크 경로 검증
- `[✅]` 내용 유실 없이 전체 워딩 마이그레이션 확인

> [!TIP]
> 이제 Claude 등 타 개발 에이전트를 사용하실 때 가장 먼저 **"AGENTS.md를 기반으로 PRD.md 등 연결된 문서를 읽고 규칙을 파악해줘"** 라고 프롬프팅하시면 AI가 스스로 올바르게 문서 트리를 순회하면서 요구사항을 가장 정확하게 인지하게 됩니다.
