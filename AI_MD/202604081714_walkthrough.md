# 작업 내역 요약 - 2026-04-08 17:14

## 목적
- NestJS의 @Cron 데코레이터를 이용하던 스케줄링을 개선하여, `.env` 파일에 지정된 환경 변수에 따라 스케줄 시간이 동기화 되도록 수정.

## 주요 변경 사항

### 1. `.env` 환경 변수 추가
- `POLLING_CRON_TIME` 변수를 `.env` 파일에 추가하였습니다. 
- 기본 동작과 동일하게 10초 주기(`*/10 * * * * *`)를 기본값으로 세팅했습니다.

### 2. BlockService 내 크론 동작 방식 변경
- `src/block/service/block.service.ts` 에서 `@Cron` 어노테이션에 의존하던 정적인 스케줄 방식을 `SchedulerRegistry` 와 `CronJob`을 활용한 **동적 등록(Dynamic Scheduling)** 구조로 변경했습니다.
- `setupPollingCron()` 메서드를 추가하여 `onModuleInit()` 단계에서 `ConfigService`를 통해 `POLLING_CRON_TIME` 값을 가져와 동적으로 스케줄러를 등록하도록 수정되었습니다.

### 3. 검증
- 코드 수정 후 정상적으로 타입 컴파일 및 빌드( `npm run build` )가 진행되는 것을 확인했습니다.
- `SchedulerRegistry` 등록 절차가 `onModuleInit` 에 성공적으로 추가되어 배포 시 오류 없이 의도대로 동적 크론 주기를 가져올 수 있습니다.
