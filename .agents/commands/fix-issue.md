# /project:fix-issue — 이슈 분석 및 수정

`$ARGUMENTS`에 이슈 번호나 오류 메시지를 전달한다.

## 실행 절차

1. **에러 컨텍스트 수집**
   - 전달된 에러 메시지 또는 이슈 내용 분석
   - 관련 파일 특정: `src/blockchain/`, `src/indexer/`
   - 최근 변경 이력 확인:
     ```bash
     git log --oneline -10
     ```

2. **원인 분석**
   - 에러 스택트레이스에서 파일과 라인 번호 추출
   - 해당 코드 위치 확인
   - 블록체인 관련 오류인 경우:
     - RPC 연결 문제인가? (네트워크/인증)
     - BigInt 직렬화 문제인가?
     - 블록 범위 초과인가?
   - TypeORM 관련 오류인 경우:
     - 엔티티 컬럼 타입 불일치인가?
     - 마이그레이션 누락인가?
   - 응답/예외 관련 오류인 경우:
     - `BusinessException` + `ErrorCode` 누락인가?
     - `GlobalExceptionsFilter` 등록이 누락되었는가?

3. **수정 계획 수립**
   수정 전 반드시 계획을 먼저 설명하고 승인을 받는다:
   - 수정할 파일 목록
   - 변경 내용 요약
   - 예상 사이드 이펙트

4. **수정 적용**
   - 코드 수정
   - 관련 테스트 업데이트 또는 신규 테스트 추가
   - 포맷 확인: `npm run format`

5. **검증**
   ```bash
   npm test
   npm run lint
   ```
